# App Store Optimization Strategy

Last updated: 2026-06-19

App working name: **Sudoku Offline**

This document defines the ASO strategy for the Sudoku mobile app. It should be
used by future agents when creating store listings, screenshots, app metadata,
release notes, landing pages, ad copy, and experiments.

The product promise is:

> A clean, offline-first Sudoku app with no ads while you play.

Important: if ads are included, do **not** describe the app as "ad-free".
Use wording like:

> No ads while solving.

or:

> No ads during gameplay.

# Official Store Metadata Limits

Verify these before final submission because store rules can change.

## Apple App Store

Current official limits:

- App name: 30 characters
- Subtitle: 30 characters
- Promotional text: 170 characters
- Description: 4000 characters
- Keywords: 100 bytes, comma-separated, no spaces unless part of a phrase

Official references:

- https://developer.apple.com/app-store/product-page/

# Positioning

## Primary positioning

**Sudoku Offline** is a calm, clean Sudoku app designed to work anywhere.

Core differentiators:

- Works offline
- No ads while playing
- Clean classic Sudoku
- Daily puzzles
- Pencil notes
- Hints
- Stats and streaks
- Dark mode
- Fast, distraction-free gameplay

## Target users

Primary users:

- Casual Sudoku players
- People searching for offline puzzle games
- Users annoyed by intrusive ads
- Daily puzzle/streak users
- Older users who want a readable, simple board
- Commuters/travellers with unreliable internet

## Main promise

Use this often:

> Classic Sudoku that works offline, with no ads while you play.

## Tone

The brand tone should be:

- Calm
- Clear
- Trustworthy
- Uncluttered
- Non-gimmicky
- Respectful

Avoid:

- Casino-like language
- Overhyped claims
- Fake urgency
- Emoji-heavy copy
- "Best", "#1", "top-rated" unless provably allowed and sourced
- Misleading "ad-free" claims if ads exist anywhere

- calm sudoku

## Apple keyword field principles

Apple has a dedicated 100-byte keyword field.

Rules:

- Do not duplicate words already in app name/subtitle unless testing proves value.
- Do not include competitor app names.
- Do not use plurals where singular is already included.
- Do not use spaces after commas.
- Do not include "app" or category names unless strategically useful.
- Do not keyword-stuff visible copy.

Example Apple keyword field if app name is `Sudoku Offline`:

```text
classic,daily,puzzle,number,logic,brain,notes,hints,wifi,relaxing,grid
```

Example if app name is `QuietGrid: Offline Sudoku`:

```text
classic,daily,puzzle,number,logic,brain,notes,hints,wifi,relaxing
```

## Google Play keyword principles

Google Play has no separate keyword field.

Use natural repetition in:

- Title
- Short description
- Full description
- Screenshot captions

Do not create word blocks like:

```text
sudoku, sudoku puzzle, sudoku offline, sudoku daily, sudoku classic...
```

Google explicitly discourages keyword stuffing and irrelevant word lists.

# Icon Strategy

The app icon should communicate:

- Sudoku/grid
- Calm
- Offline/focused
- Premium simplicity

Preferred visual concept:

- Rounded square
- Soft 3x3 Sudoku grid
- One highlighted cell
- No tiny numbers
- No text
- Calm blue/indigo/cream palette

Avoid:

- Busy full 9x9 grid
- Tiny unreadable numbers
- Wi-Fi-off symbol as the main mark unless tested
- Cartoon mascots
- Casino-style colours
- Text inside the icon

Icon variants to test later:

1. Cream background, indigo grid, gold highlighted cell
2. Dark navy background, blue grid, warm highlighted cell
3. Minimal grid with one pencil-mark dot pattern
4. More literal 9x9 grid cropped into icon

# Review Prompt Strategy

Do not ask for a rating immediately.

Good moments:

- After completing 3 puzzles
- After completing a daily puzzle with no crash
- After a 3-day streak
- After a mistake-free completion
- After user voluntarily opens stats or shares a result

Bad moments:

- On first launch
- After losing progress
- After a mistake
- After an ad
- After purchase failure
- While playing

Prompt copy:

```text
Enjoying Sudoku Offline?
A quick rating helps more players find the app.
```

# Metrics To Track

## Store metrics

Track separately for Apple and Google:

- Impressions
- Product page views
- Install conversion rate
- Source/medium
- Keyword performance if available
- Country performance
- Screenshot experiment results
- Custom page/listing conversion rate

## Product metrics

ASO quality depends on post-install quality.

Track:

- D1 retention
- D7 retention
- D30 retention
- Puzzle started
- Puzzle completed
- Daily puzzle completed
- Average puzzles per user
- Average session length
- Crash-free sessions
- Rating prompt shown
- Rating prompt accepted
- Remove-ads purchase conversion
- Ad impressions per active user
- Reviews mentioning ads

## Success targets

Initial targets:

- Store conversion rate: improve over baseline through experiments
- App rating: 4.5+
- Crash-free sessions: 99.5%+
- D1 retention: 25%+
- D7 retention: 8%+
- Reviews should mention "offline", "clean", "simple", "no ads while playing"

# Agent Rules

When future agents modify store copy, screenshots, onboarding, or monetisation,
they must follow these rules:

1. Do not claim the app is fully ad-free if ads exist.
2. Do not add ads during gameplay.
3. Do not weaken the offline promise.
4. Do not keyword-stuff descriptions.
5. Do not use competitor names in metadata.
6. Do not use "best", "#1", "top", or award claims without proof.
7. Do not use excessive emoji or special characters in Google Play metadata.
8. Keep the first screenshot focused on the strongest value proposition.
9. Keep copy simple and readable.
10. Prioritize conversion quality, not just installs.
11. Monitor reviews for complaints about ads, crashes, readability, and lost progress.
12. Update this document after meaningful ASO experiments.
